# Sprint Review and Retrospective Agenda

## Review

1. Demo completed work.
2. Check sprint goal outcome.
3. Review model metrics or documentation evidence.
4. Confirm what is ready for submission or presentation.

## Retrospective

Answer three questions:

1. What worked?
2. What slowed us down?
3. What will we change next sprint?

## Output

- Review notes in Confluence
- Retro actions added to Jira
- Sprint closed
