# Team Charter

## Team Name

**CourtVision AI**

## Why This Name

CourtVision AI communicates both tennis domain focus and analytical clarity. It is concise, presentation-friendly, and suitable for a leadership-facing hackathon project.

## Goal

Win the EXL Analytics & AI Hackathon 2026 by delivering a strong Wimbledon prediction system with clean data, robust modeling, explainable recommendations, and a polished final narrative.

## Working Agreement

- Keep Jira current before every standup.
- Use GitHub branches for all code and documentation changes.
- Make decisions in writing, preferably in Confluence.
- Prefer small pull requests over large late merges.
- Treat model explanations and reproducibility as first-class deliverables.
- Raise blockers early, especially because the team is split between the USA and India.

## Roles

| Person | Primary Role | Secondary Role |
| --- | --- | --- |
| Rohit Kumar | Team Lead and Product Owner | Presentation, prioritization, stakeholder story |
| Abhinay Singh | Data Engineering | Dataset discovery, feature reliability, data documentation |
| Harshit Singh | Modeling and Evaluation | Model experiments, metrics, validation, explainability |

## Time Zone Operating Rule

Use India Standard Time as the primary project clock because hackathon deadlines are in IST. Rohit should post end-of-day USA updates that become next-morning India work items.

## Decision Rule

Rohit owns final priority calls. Technical choices should be decided by evidence from validation metrics, model interpretability, and submission risk.
